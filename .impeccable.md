## Design Context

### Users
- **Who**: macOS developers using MiniMax API for coding tasks
- **Context**: Menu bar utility - glanceable, unobtrusive, accessed multiple times daily
- **Job**: Monitor API usage quota (daily 5-hour window + weekly) at a glance without opening a browser
- **Emotional goal**: Confidence - know exactly how much quota remains; avoid unexpected API blocks

### Brand Personality
- **Voice**: Technical, precise, utilitarian
- **Tone**: Clean, no-nonsense, efficient
- **3-word personality**: Minimal · Informative · Developer-focused
- **Emotional goal**: Calm confidence - user always knows their quota status

### Aesthetic Direction
- **Visual tone**: macOS native, system-consistent, no custom decorations
- **Theme**: Light/Dark mode aware, uses system colors where possible
- **Anti-reference**: No flashy animations, no gradients, no unnecessary visual elements

### Design Principles
1. **System consistency** - Use native macOS controls and spacing; avoid reinventing UI patterns
2. **Glanceability** - Menu bar shows critical info (percentage) in smallest possible footprint; detail in popover
3. **Clear hierarchy** - Primary action/status visible, secondary info accessible but not dominant
4. **Functional color** - Colors convey meaning (green/yellow/red = usage level), not decoration
5. **Resilient input** - TextField must handle Cmd+V paste in NSPopover context (known macOS limitation)
